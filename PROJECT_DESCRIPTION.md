# Minecraft P2P Connector

**Minecraft P2P Connector** — это легковесное, кроссплатформенное приложение (Windows и Android), предназначенное для соединения игроков в Minecraft (Java и Bedrock Edition) напрямую по принципу P2P (Peer-to-Peer), как если бы они находились в одной локальной сети (LAN).

Оно избавляет от необходимости вручную "пробрасывать" (port forwarding) порты на роутере, покупать хостинг или использовать сторонние VPN-сервисы вроде Radmin VPN или Hamachi.

---

## 🛠 Технологии и Стек

Проект использует современный технологический стек, разделенный на мощный бэкенд и легковесный, быстрый фронтенд:

### Бэкенд
* **Язык**: [Rust](https://www.rust-lang.org/) — гарантирует максимальную производительность, безопасную работу с памятью и сетями.
* **Фреймворк приложения**: [Tauri v2](https://v2.tauri.app/) — кроссплатформенный фреймворк, позволяющий собрать бэкенд на Rust и фронтенд на веб-технологиях в единое нативное приложение для Desktop (Windows) и Mobile (Android).
* **Сеть (P2P)**: `quinn` (библиотека [QUIC](https://en.wikipedia.org/wiki/QUIC) на Rust) — обеспечивает зашифрованное, быстрое (на базе UDP) и мультиплексированное соединение напрямую между игроками.
* **NAT Traversal (Прохождение NAT)**: 
  * `rust-stun` — для определения внешнего IP и порта (STUN) и анализа типа NAT.
  * `rupnp` — для автоматического проброса портов через роутер по протоколу UPnP в качестве резервного или улучшающего варианта.
* **Поиск миров (Discovery)**:
  * Встроенные UDP-слушатели (RakNet) для перехвата LAN-широковещательных пакетов Bedrock (порт 19132) и Java серверов. 
  * Прямое чтение файлов из папок Minecraft для автоопределения имени мира Bedrock.

### Фронтенд (Интерфейс)
* **Технологии**: Vanilla JavaScript (ES6+), HTML5, CSS3. Отсутствие тяжелых UI-фреймворков вроде React/Vue обеспечивает минимальное потребление оперативной памяти и мгновенный запуск.
* **Сборщик**: [Vite](https://vitejs.dev/) — для быстрой сборки и HMR (Hot Module Replacement) во время разработки.
* **Сигналинг и Лобби (Signaling)**: [Ably Realtime](https://ably.com/) — используется по протоколу WebSocket для "знакомства" хоста и клиента. Хост публикует информацию о своей комнате, а клиент находит ее в "Лобби" и получает нужные параметры (STUN/ICE-кандидаты) для прямой установки QUIC-соединения.

## 🔑 Ключи подключения (Signaling)

Для того чтобы устройства могли найти друг друга в интернете *до* установки прямого P2P соединения, используется внешний Signaling-сервер (Ably).

В проекте зашит следующий бесплатный API ключ от **Ably**:
```javascript
const ABLY_API_KEY = "aGkPAA.1VHkjw:Bai-67g05FcqHdfVOMiSfjYlK3aLz8wOzj5WeTgz4cw";
const LOBBY_CHANNEL_NAME = "minecraft-lobby";
```

**Как это работает:**
1. **Хост** подключается к каналу `minecraft-lobby` в Ably.
2. Он публикует (Broadcast) туда информацию о своей комнате, свой внешний IP:Port (полученный от STUN) и версию игры.
3. **Клиент** слушает этот же канал. При появлении новой комнаты она отображается в интерфейсе клиента.
4. При клике на "Подключиться", клиент берет внешний IP:Port из канала Ably и инициирует **прямое (P2P) UDP/QUIC** соединение с хостом.
5. *После успешного соединения Ably больше не используется* — весь игровой трафик идет напрямую от игрока к игроку.

## 🚀 Основной функционал

1. **Host Mode (Режим Хоста)**:
   * Автоматически находит запущенные LAN-миры на устройстве хоста (поддерживается и Java и Bedrock).
   * Автоматически извлекает название Bedrock мира (читая файл `levelname.txt` в папке игры) для красивого отображения в лобби.
   * Открывает P2P UDP-туннель и публикует сервер в лобби (Ably).

2. **Client Mode (Режим Клиента)**:
   * Подключается к P2P-туннелю хоста.
   * Локально симулирует работу сервера (создавая фантомный сервер на устройстве клиента).
   * Отправляет "широковещательные" пакеты в сеть клиента, благодаря чему игра думает, что сервер находится в локальной домашней сети (появляется во вкладке "Друзья" или "Локальная сеть" прямо в Minecraft).

3. **Инструменты диагностики**:
   * Встроенный тест NAT: определяет, какой тип соединения используется (Symmetric, Cone NAT и т.д.), чтобы понимать, возможен ли прямой пробив (UDP Hole Punching).
   * Диагностика пинга и проверка STUN-доступа.
   * Тестовый встроенный сервер: позволяет развернуть фейковый сервер прямо в приложении для тестов маршрутизации, не запуская сам Minecraft.

4. **Кроссплатформенность**:
   * Полная поддержка Windows Desktop.
   * Поддержка Android (включает встроенный `.so` бинарник с JNI-прослойкой для работы Rust на мобильных устройствах).

## 📂 Структура проекта

```text
android p2p/       # Android-версия (APK собирается через tauri android build)
p2p/               # Основная кодовая база (Windows/Desktop)
 ├─ src/           # Фронтенд: HTML, CSS, JS 
 │   ├─ index.html # Главный экран интерфейса
 │   ├─ main.js    # Логика UI, Ably, общение с Tauri
 │   ├─ i18n.js    # Система мультиязычности (RU/EN)
 │   └─ styles.css # Стили
 ├─ src-tauri/     # Бэкенд: Rust
 │   ├─ src/
 │   │   ├─ main.rs          # Точка входа Tauri (для Windows)
 │   │   ├─ lib.rs           # Точка входа Tauri (для Android)
 │   │   ├─ network/         # Сетевая логика P2P
 │   │   │   ├─ tunnel.rs    # Управление QUIC P2P туннелями
 │   │   │   ├─ upnp.rs      # Логика проброса портов UPnP
 │   │   │   ├─ stun.rs      # NAT определение
 │   │   │   └─ discovery.rs # Широковещание Minecraft LAN 
 │   ├─ tauri.conf.json      # Настройки сборки и доступов Tauri
 │   └─ Cargo.toml           # Зависимости Rust
```
